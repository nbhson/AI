# Rule: Template Selection

Always use the designated markdown templates for generating investigation reports, technical designs, and PR summaries to ensure consistency across the TOEIC application workflows.

| **Phase**                    | **Template**                     | **When to Use**                                                           |
| ---------------------------- | -------------------------------- | ------------------------------------------------------------------------- |
| **Ticket Review**            | `jira-ticket-review-template.md` | Reviewing requirements, acceptance criteria, and initial scope from Jira. |
| **Investigation**            | `bug-report-template.md`         | Analyzing bug symptoms and root causes during the bug-fix workflow.       |
| **Planning**                 | `technical-design-template.md`   | Designing technical solutions for feature-delivery or refactor workflows. |
| **Code Review**              | `pull-request-template.md`       | Submitting code changes or performing peer reviews.                       |
| **Implementation Completed** | `engineering-report-template.md` | Summarizing changes, validation results, and risks after implementation.  |
